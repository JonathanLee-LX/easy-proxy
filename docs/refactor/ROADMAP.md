# Easy Proxy 功能规划

> 产品定位：面向前端/全栈开发者的轻量级开发代理工具，核心价值 —— **简单、够用、开箱即用**。

## 重构进展（插件架构 Phase 1）

- 已完成插件运行时骨架（`PluginManager` + `HookDispatcher`）与 `off/shadow/on` pipeline
- 已完成内置 `router`/`logger` 插件接线，`mock` 插件支持 inline 短路（默认关闭）
- 已提供重构观测 API：`/api/pipeline/*`、`/api/plugins*`、`/api/refactor/status`
- 已建立重构回归基线（`npm run test:refactor`）与 smoke 验证脚本（`npm run smoke:refactor`）
- 下一阶段重点：按 host 白名单小流量 `on` 模式灰度 + 回滚演练

## 待开发功能

### 第一梯队：核心开发体验提升

#### 1. 请求耗时 & 状态码显示

在日志表格中增加 `statusCode` 和 `duration`（响应耗时）列，用颜色区分状态码（2xx 绿 / 3xx 蓝 / 4xx 黄 / 5xx 红）。

- **实现成本**：🟢 低（~0.5 天）
  - 后端：在代理请求前后记录时间戳，将 `statusCode` 和 `duration` 追加到 `logData`
  - 前端：`ProxyRecord` 类型增加两个字段，`LogTable` 增加两列，加颜色样式
- **收益**：🔴 高
  - 状态码和耗时是调试最基本的信息，当前完全缺失
  - 几乎零学习成本，所有开发者都需要
- **优先级**：⭐⭐⭐⭐⭐

---

#### 2. 响应延迟模拟（Throttle / Latency）

对匹配规则的请求添加可配置的延迟（如 500ms / 2000ms / 5000ms），模拟慢速网络或后端慢响应。

- **实现成本**：🟢 低（~1 天）
  - 后端：在 Mock 规则中增加 `delay` 字段，发送响应前 `setTimeout`；可选全局节流配置
  - 前端：Mock 编辑表单增加延迟输入框
- **收益**：🔴 高
  - 测试 loading 状态、骨架屏、超时处理、竞态条件的刚需
  - 无需外部工具（Chrome 自带节流粒度太粗）
- **优先级**：⭐⭐⭐⭐⭐

---

#### 3. 请求重放（Replay）

从日志详情中一键重新发送请求，并展示新的响应结果。

- **实现成本**：🟢 低（~1 天）
  - 后端：新增 `POST /api/replay/:id` 接口，使用已存储的 requestHeaders + requestBody 重发请求
  - 前端：详情面板增加"重放"按钮，展示重放结果
- **收益**：🔴 高
  - 免去反复在浏览器触发操作来复现请求
  - 接口调试效率大幅提升，尤其是 POST/PUT 等非幂等请求
- **优先级**：⭐⭐⭐⭐⭐

---

#### 4. Map Local（映射本地文件）

将远程 URL 映射到本地文件路径，返回本地文件内容。典型场景：线上页面替换 JS/CSS 为本地构建产物。

- **实现成本**：🟡 中（~2 天）
  - 后端：扩展规则系统支持 `file://` 协议的 target，读取本地文件并返回，处理 MIME 推断
  - 前端：规则配置中支持选择本地文件路径或输入路径，增加 Map Local 类型标识
  - 边界：需处理文件不存在、大文件、二进制文件等情况
- **收益**：🔴 高
  - Charles "Map Local" 是最常用的调试功能之一
  - 对于线上环境调试本地代码几乎是唯一方案
- **优先级**：⭐⭐⭐⭐

### 第二梯队：效率与协作

#### 5. 高级过滤

支持按域名、状态码范围、HTTP 方法、协议（h2/h1.1）、是否 Mock 等多维度筛选日志。

- **实现成本**：🟡 中（~1.5 天）
  - 前端为主：设计筛选 UI（下拉 / 标签 / 快捷按钮），扩展 `useFuzzyFilter` 支持多条件组合
  - 依赖功能 1（状态码字段），否则无法按状态码筛选
- **收益**：🟡 中高
  - 请求量大时仅靠模糊搜索效率低
  - 快速定位 4xx/5xx 错误请求的刚需
- **优先级**：⭐⭐⭐⭐

---

#### 6. 规则导入/导出 & 团队共享

一键导出代理规则 + Mock 规则为 JSON 文件，支持导入覆盖或合并。

- **实现成本**：🟢 低（~0.5 天）
  - 后端：新增 `GET /api/export` 返回完整配置 JSON，`POST /api/import` 接收并写入
  - 前端：规则页和 Mock 页各加导出/导入按钮，用浏览器 File API 实现上传下载
- **收益**：🟡 中
  - 团队统一调试环境，新成员快速上手
  - 但单人使用场景下价值有限
- **优先级**：⭐⭐⭐

---

#### 7. HAR 格式导出

将请求日志导出为标准 HAR (HTTP Archive) 格式，可导入 Chrome DevTools 或其他抓包工具。

- **实现成本**：🟡 中（~1.5 天）
  - 后端：将 `proxyRecordArr` + `proxyRecordDetailMap` 转换为 HAR 1.2 规范的 JSON 结构
  - 前端：日志页增加"导出 HAR"按钮
  - 需要仔细处理 HAR 规范中的 timings、cookies、queryString 等字段
- **收益**：🟡 中
  - 与后端/测试同事共享问题请求的标准方式
  - 可利用现有 HAR 分析工具做进一步分析
- **优先级**：⭐⭐⭐

---

#### 8. 环境快速切换（Web UI）

在 Web 界面直接列出 `.epconfig` 目录下所有配置环境，点击切换无需重启。

- **实现成本**：🟡 中（~1.5 天）
  - 后端：新增 `GET /api/envs` 扫描 `.epconfig` 目录返回可用环境列表，`PUT /api/envs/switch` 重新加载指定配置
  - 前端：Header 区域或规则页增加环境选择器下拉
  - 需处理配置切换后的热更新、当前环境状态展示
- **收益**：🟡 中
  - 多环境频繁切换时省去重启的麻烦
  - 适合同时调试多个后端环境的场景
- **优先级**：⭐⭐⭐

---

### 第三梯队：体验完善

#### 9. 请求断点 & 修改（Breakpoints）

设置断点规则，匹配的请求在发出前/响应返回前暂停，允许用户实时编辑 Headers、Body、状态码后再放行。

- **实现成本**：🔴 高（~4-5 天）
  - 后端：实现请求队列 + 挂起机制，通过 WebSocket 双向通信等待前端编辑结果，超时自动放行
  - 前端：断点规则管理 UI、实时编辑弹窗、请求队列展示
  - 复杂度：并发请求排队、超时处理、连接保持、断点规则 CRUD
- **收益**：🔴 高
  - Charles/Fiddler 的杀手级功能，能动态修改真实请求和响应
  - Mock 只能返回固定内容，断点可以在真实响应基础上微调
- **优先级**：⭐⭐⭐（收益高但成本也高，建议核心功能稳定后再做）

---

#### 10. WebSocket 消息日志

记录 WebSocket 连接中的每一帧消息（上行/下行），支持时间线查看和内容搜索。

- **实现成本**：🟡 中偏高（~2-3 天）
  - 后端：在 WebSocket 代理的 `on('message')` 中记录帧数据，设计存储结构（连接维度 + 消息列表）
  - 前端：新增 WebSocket 日志标签页，展示连接列表和消息时间线
  - 需处理二进制帧、大消息截断、消息量控制
- **收益**：🟡 中
  - 对实时通信类应用（IM、协同编辑、实时数据推送）调试价值大
  - 但非所有项目都用 WebSocket，受众偏窄
- **优先级**：⭐⭐

---

#### 11. 证书信任引导

首次使用时检测 Root CA 是否已被系统信任，未信任则展示平台相关的图文引导。

- **实现成本**：🟡 中（~1 天）
  - 后端：检测证书信任状态（macOS: `security verify-cert`，Windows: `certutil`）
  - 前端：首次打开时展示引导弹窗，含一键信任按钮（macOS 可调用 `security add-trusted-cert`）
  - 需适配 macOS / Windows / Linux 三平台
- **收益**：🟡 中
  - 降低 HTTPS 代理的使用门槛
  - 但属于一次性操作，后续不再触发
- **优先级**：⭐⭐

---

#### 12. 暗黑模式

Web 界面支持 dark / light 主题切换。

- **实现成本**：🟢 低（~0.5 天）
  - 项目已使用 shadcn/ui + Tailwind CSS，天然支持 `dark:` 类
  - 仅需添加主题切换按钮、ThemeProvider、localStorage 持久化
- **收益**：🟢 低
  - 提升长时间使用的舒适度
  - 对功能无实质帮助，属于锦上添花
- **优先级**：⭐⭐
